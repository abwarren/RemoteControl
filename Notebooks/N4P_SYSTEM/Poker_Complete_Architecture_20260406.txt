## System 1: Remote Control (Manual Player Control)

### Architecture
```
UI (HTML/JS) ←→ Flask Backend ←→ n4p.js (injected in browser)

  User clicks      Command Queue      DOM automation

  - MIN ALL, MAX ALL, BET/EXECUTE, CASHOUT ALL
- **Pre-action checkboxes:**
  - Check/Fold, Check/Call (mutually exclusive)
- **Command log:** Last 20 commands
- **Real-time updates:** 2-second polling

## System 2: Collector Agent (Automated Card Scraping)

### Architecture (RDS Version - Recommended)
```
┌──────────────────────────────────────────────────┐
│   EC2: 9 Selenium Bots                           │
│   - Persistent browser sessions                  │
│   - DOM polling: icon-layer2 cards              │
│   - POST snapshots every 1-2s                    │
└────────────┬─────────────────────────────────────┘
			 │
			 ▼
┌──────────────────────────────────────────────────┐
│   API Gateway → Lambda → RDS Proxy → PostgreSQL │
│   - Validate/normalize cards                     │
│   - Upsert table_state (latest)                 │
│   - Insert snapshot_history (log)               │
│   - Archive to S3 (raw JSON)                    │
└──────────────────────────────────────────────────┘
			 │
			 ▼
┌──────────────────────────────────────────────────┐
│   PostgreSQL Tables:                             │
│   - table_state (latest row per table)          │
│   - snapshot_history (append-only log)          │

└──────────────────────────────────────────────────┘
			 │

### Why RDS?
✅ **Relational data model** (bots → tables → snapshots)  
✅ **SQL queries, joins, aggregations**  
✅ **Full history and replay**  
✅ **Easier debugging**  
⚠️ **Requires RDS Proxy** (prevent Lambda connection storms)

## Key Design Decisions

### Decision 1: Lambda + RDS (Not Flask In-Memory)
**Why:**
- Serverless auto-scaling
- RDS connection pooling via RDS Proxy
- S3 archival for history/replay
- Cheaper than EC2 for processing layer

**Cost:** ~$138/month (RDS $28 + Lambda $9 + S3 $101)

### Decision 2: RDS (Not DynamoDB)
**Why:**
- Relational data model (bots, tables, snapshots, commands)
- Need for SQL queries, joins, history
- DynamoDB = key-value store (not ideal for complex queries)

**Trade-off:** $7/month more expensive than DynamoDB

### Decision 3: EC2 for Bots (Not Lambda)
**Why:**
- Lambda can't run persistent browser sessions
- Need logged-in state, iframe navigation
- Continuous DOM polling (not short-lived)

**Bots Stay on EC2, Backend Moves to Lambda + RDS**

## Security Groups

### 1. EC2 Bots Security Group

**Name:** `sg-ec2-bots`

**Inbound:**
- SSH (22) from your IP only (for management)

**Outbound:**
- HTTPS (443) to 0.0.0.0/0 (API Gateway, pokerbet.co.za)
- HTTP (80) to 0.0.0.0/0 (if needed)

### 2. Lambda Security Group

**Name:** `sg-lambda-ingest`

**Inbound:**
- None (Lambda doesn't receive inbound)

**Outbound:**
- PostgreSQL (5432) to `sg-rds-proxy`
- HTTPS (443) to 0.0.0.0/0 (S3 access)

### 3. RDS Proxy Security Group

**Name:** `sg-rds-proxy`

**Inbound:**
- PostgreSQL (5432) from `sg-lambda-ingest`

**Outbound:**
- PostgreSQL (5432) to `sg-rds`

### 4. RDS Security Group

**Name:** `sg-rds`

**Inbound:**
- PostgreSQL (5432) from `sg-rds-proxy` ONLY

**Outbound:**
- None (RDS doesn't initiate outbound)

**Key Security Principle:**
```
EC2 bots  →  (HTTPS)  →  API Gateway  →  Lambda  →  RDS Proxy  →  RDS
										 (SG allowed)  (SG allowed)
```

**RDS is NOT directly accessible:**
- No public IP
- No Lambda → RDS direct connection
- Only RDS Proxy → RDS

### Phase 3: RDS Setup

#### 3.1 Create DB Subnet Group

```bash
aws rds create-db-subnet-group \
  --db-subnet-group-name poker-db-subnet-group \
  --db-subnet-group-description "Subnet group for poker RDS" \
  --subnet-ids subnet-db-1a-xxxxxxxx subnet-db-1b-xxxxxxxx \
  --tags Key=Name,Value=poker-db-subnet-group
```

#### 3.2 Create RDS Instance

```bash
aws rds create-db-instance \
  --db-instance-identifier poker-collector-db \
  --db-instance-class db.t3.micro \
  --engine postgres \
  --engine-version 14.7 \
  --master-username postgres \
  --master-user-password 'SECURE_PASSWORD_HERE' \
  --allocated-storage 20 \
  --storage-type gp3 \
  --vpc-security-group-ids sg-rds \
  --db-subnet-group-name poker-db-subnet-group \
  --backup-retention-period 7 \
  --preferred-backup-window "03:00-04:00" \
  --preferred-maintenance-window "mon:04:00-mon:05:00" \
  --no-publicly-accessible \
  --tags Key=Name,Value=poker-collector-db
```

**Wait for instance to be available (~10 minutes):**
```bash
aws rds wait db-instance-available --db-instance-identifier poker-collector-db
```

#### 3.3 Create RDS Proxy

**First, store credentials in Secrets Manager:**
```bash
aws secretsmanager create-secret \
  --name poker-db-credentials \
  --description "RDS credentials for poker collector" \
  --secret-string '{"username":"postgres","password":"SECURE_PASSWORD_HERE"}'
```

**Create IAM role for RDS Proxy:**
```json
{
  "Version": "2012-10-17",
  "Statement": [
	{
	  "Effect": "Allow",
	  "Principal": {
		"Service": "rds.amazonaws.com"
	  },
	  "Action": "sts:AssumeRole"
	}
  ]
}
```

**Create RDS Proxy:**
```bash
aws rds create-db-proxy \
  --db-proxy-name poker-collector-proxy \
  --engine-family POSTGRESQL \
  --auth AuthScheme=SECRETS,SecretArn=arn:aws:secretsmanager:...:secret:poker-db-credentials \
  --role-arn arn:aws:iam::ACCOUNT_ID:role/RDSProxyRole \
  --vpc-subnet-ids subnet-db-1a-xxxxxxxx subnet-db-1b-xxxxxxxx \
  --vpc-security-group-ids sg-rds-proxy \
  --require-tls
```

**Register RDS instance with proxy:**
```bash
aws rds register-db-proxy-targets \
  --db-proxy-name poker-collector-proxy \
  --db-instance-identifiers poker-collector-db
```

**Get proxy endpoint:**
```bash
aws rds describe-db-proxies --db-proxy-name poker-collector-proxy \
  --query 'DBProxies[0].Endpoint' --output text

# Output: poker-collector-proxy.proxy-xxxxxxxx.us-east-1.rds.amazonaws.com
```

# Collector Agent - RDS Architecture (Lambda + RDS Proxy + PostgreSQL)
**Optimal Relational Backend for Poker Data Collection**

## Why RDS Instead of DynamoDB?

Your data model is **naturally relational**:
- `bots` → many `tables` → many `snapshots`
- `tables` → many `commands`
- `sessions` → many `errors`
- Need for **joins**, **history queries**, **aggregations**

✅ **RDS Postgres** = designed for structured relational workloads  
⚠️ **DynamoDB** = key-value store (not ideal for complex queries)

## Why RDS Proxy?

**Problem:** Lambda creates new DB connection on each invocation  
**At Scale:** 15M Lambda invocations/month = potential connection storm  
**Solution:** RDS Proxy pools connections, reuses them across Lambda invocations

**Without RDS Proxy:**
```
Lambda1 → RDS (new connection)
Lambda2 → RDS (new connection)
Lambda3 → RDS (new connection)
...
Lambda1000 → RDS (ERROR: too many connections)
```

**With RDS Proxy:**
```
Lambda1 →┐
Lambda2 →├─→ RDS Proxy (connection pool) → RDS (5 connections)
Lambda3 →┤
Lambda4 →┘
```

## 1. PostgreSQL Schema

### 1.1 Table: `table_state` (Latest State Per Table)

```sql
CREATE TABLE table_state (
	table_id VARCHAR(64) PRIMARY KEY,
	source_bot VARCHAR(64) NOT NULL,
	hero_cards TEXT[],                 -- Array of cards: ['Ah', 'Kh']
	board_cards TEXT[],                -- Array of cards: ['Qh', 'Jh', 'Th']
	raw_cards TEXT[],                  -- All cards in order
	status VARCHAR(16) NOT NULL,       -- 'LIVE', 'WAITING', 'STALE'
	last_seen BIGINT NOT NULL,         -- Unix timestamp
	updated_at TIMESTAMP DEFAULT NOW(),
    
	-- Indexes
	INDEX idx_status (status),
	INDEX idx_source_bot (source_bot),
	INDEX idx_last_seen (last_seen DESC)
);

-- Auto-update updated_at on every UPDATE
CREATE TRIGGER update_table_state_updated_at
BEFORE UPDATE ON table_state
FOR EACH ROW
EXECUTE FUNCTION update_updated_at_column();

CREATE FUNCTION update_updated_at_column()
RETURNS TRIGGER AS $$
BEGIN
	NEW.updated_at = NOW();
	RETURN NEW;
END;
$$ LANGUAGE plpgsql;
```

**Purpose:**
- Fast lookups by `table_id` (primary key)
- Query latest state for UI
- One row per table (upsert on conflict)

**Example Query:**
```sql
-- Get all active tables
SELECT * FROM table_state WHERE status = 'LIVE' ORDER BY last_seen DESC;

-- Get specific table
SELECT * FROM table_state WHERE table_id = 'bot_1';
```

	-- Indexes

	INDEX idx_timestamp (timestamp_unix DESC),
	INDEX idx_created_at (created_at DESC)
);

-- Partition by month for performance (optional, for high volume)
-- CREATE TABLE snapshot_history_2026_04 PARTITION OF snapshot_history

**Purpose:**

CREATE TABLE bot_registry (

CREATE TABLE commands (

	command_value NUMERIC(10, 2),      -- Optional: bet amount, etc.

## 2. Lambda Function (Python + psycopg2)

### 2.1 Lambda Handler

```python
import json
import os
import psycopg2
import boto3
from datetime import datetime

# Environment variables
RDS_PROXY_ENDPOINT = os.environ['RDS_PROXY_ENDPOINT']
DB_NAME = os.environ['DB_NAME']
DB_USER = os.environ['DB_USER']
DB_PASSWORD = os.environ['DB_PASSWORD']  # From Secrets Manager
S3_BUCKET = os.environ['S3_BUCKET']

# AWS clients
s3_client = boto3.client('s3')

# Database connection (reused across invocations)
_db_conn = None

def get_db_connection():
	"""Get or create database connection (reuses across Lambda invocations)."""
	global _db_conn
    
	if _db_conn is None or _db_conn.closed:
		_db_conn = psycopg2.connect(
			host=RDS_PROXY_ENDPOINT,
			port=5432,
			database=DB_NAME,
			user=DB_USER,
			password=DB_PASSWORD,
			connect_timeout=5
		)
    
	return _db_conn


def lambda_handler(event, context):
	"""
	Process poker table snapshot.
    
	1. Validate payload
	2. Upsert to table_state (latest)
	3. Insert to snapshot_history (log)
	4. Archive to S3
	"""
    
	# Parse snapshot
	try:
		if 'body' in event:
			snapshot = json.loads(event['body'])
		else:
			snapshot = event
	except Exception as e:
		return {
			'statusCode': 400,
			'body': json.dumps({'error': 'Invalid JSON', 'details': str(e)})
		}
    
	# Validate required fields
	required = ['table_id', 'source_bot', 'timestamp']
	for field in required:
		if field not in snapshot:
			return {
				'statusCode': 400,
				'body': json.dumps({'error': f'Missing field: {field}'})
			}
    
	table_id = snapshot['table_id']
	source_bot = snapshot['source_bot']
	timestamp = snapshot['timestamp']
	hero_cards = snapshot.get('hero_cards', [])
	board_cards = snapshot.get('board_cards', [])
	raw_cards = snapshot.get('raw_cards', [])
	status = snapshot.get('status', 'LIVE')
    
	try:
		conn = get_db_connection()
		cur = conn.cursor()
        
		# 1. Upsert to table_state (latest state)
		cur.execute("""
			INSERT INTO table_state (
				table_id, source_bot, hero_cards, board_cards, 
				raw_cards, status, last_seen
			) VALUES (%s, %s, %s, %s, %s, %s, %s)
			ON CONFLICT (table_id) DO UPDATE SET
				source_bot = EXCLUDED.source_bot,
				hero_cards = EXCLUDED.hero_cards,
				board_cards = EXCLUDED.board_cards,
				raw_cards = EXCLUDED.raw_cards,
				status = EXCLUDED.status,
				last_seen = EXCLUDED.last_seen,
				updated_at = NOW()
		""", (table_id, source_bot, hero_cards, board_cards, 
			  raw_cards, status, timestamp))
        
		# 2. Insert to snapshot_history (append-only log)
		cur.execute("""
			INSERT INTO snapshot_history (
				table_id, source_bot, hero_cards, board_cards, 
				raw_cards_json, status, timestamp_unix
			) VALUES (%s, %s, %s, %s, %s, %s, %s)
		""", (table_id, source_bot, hero_cards, board_cards,
			  json.dumps(raw_cards), status, timestamp))
        
		conn.commit()
		cur.close()
        
		# 3. Archive to S3 (async, non-blocking)
		archive_to_s3(snapshot, table_id, timestamp)
        
		return {
			'statusCode': 200,
			'body': json.dumps({
				'ok': True,
				'table_id': table_id,
				'message': 'Snapshot processed'
			})
		}
        
	except Exception as e:
		print(f"Database error: {e}")
		return {
			'statusCode': 500,
			'body': json.dumps({
				'ok': False,
				'error': 'Database write failed',
				'details': str(e)
			})
		}


def archive_to_s3(snapshot, table_id, timestamp):
	"""Archive raw snapshot to S3 (non-blocking)."""
	try:
		date = datetime.fromtimestamp(timestamp).strftime('%Y-%m-%d')
		key = f"snapshots/{table_id}/{date}/{int(timestamp)}.json"
        
		s3_client.put_object(
			Bucket=S3_BUCKET,
			Key=key,
			Body=json.dumps(snapshot),
			ContentType='application/json'
		)
        
		print(f"S3: Archived {key}")
	except Exception as e:
		print(f"S3 archive error (non-fatal): {e}")
```

## Why This Architecture?

### ✅ Use Lambda + DynamoDB + S3 For:
- **Stateless snapshot processing** (validate, normalize, store)
- **Fast live state lookups** (DynamoDB indexed by table_id)
- **Cheap raw data archival** (S3 for history/replay/debug)
- **Automatic scaling** (no server management)
- **Fan-out to analytics** (S3 triggers for downstream processing)

### ❌ Do NOT Use Lambda For:
- **Running Selenium/Chrome** (needs persistent sessions)
- **Live browser automation** (long-running processes)
- **iframe navigation** (requires logged-in state)
- **Continuous DOM polling** (Lambda is short-lived)

## 2. Lambda Function Implementation

### 2.1 Lambda Handler (Python 3.11)

**File:** `lambda_function.py`

```python
import json
import boto3
import time
from datetime import datetime

# AWS clients
dynamodb = boto3.resource('dynamodb')
s3_client = boto3.client('s3')

# Configuration
DYNAMODB_TABLE = 'poker_tables'
S3_BUCKET = 'poker-snapshots-archive'
TTL_SECONDS = 60  # Auto-delete stale tables after 60s

def lambda_handler(event, context):
	"""
	Lambda function to process poker table snapshots.
    
	Receives snapshot from API Gateway POST request.
	Validates, normalizes, stores in DynamoDB and S3.
	"""
    
	# Parse incoming snapshot
	try:
		if 'body' in event:
			# API Gateway format
			body = json.loads(event['body'])
		else:
			# Direct invocation format
			body = event
        
		snapshot = body
	except Exception as e:
		return {
			'statusCode': 400,
			'body': json.dumps({'error': 'Invalid JSON', 'details': str(e)})
		}
    
	# Validate required fields
	required = ['table_id', 'source_bot', 'timestamp']
	for field in required:
		if field not in snapshot:
			return {
				'statusCode': 400,
				'body': json.dumps({'error': f'Missing required field: {field}'})
			}
    
	table_id = snapshot['table_id']
	source_bot = snapshot['source_bot']
	timestamp = snapshot['timestamp']
    
	# Normalize cards (ensure lists)
	hero_cards = snapshot.get('hero_cards', [])
	board_cards = snapshot.get('board_cards', [])
	raw_cards = snapshot.get('raw_cards', [])
	status = snapshot.get('status', 'LIVE')
    
	# Validate card format (optional but recommended)
	if not validate_cards(hero_cards + board_cards):
		return {
			'statusCode': 400,
			'body': json.dumps({'error': 'Invalid card format'})
		}
    
	# Write to DynamoDB (latest state)
	try:
		write_to_dynamodb(
			table_id=table_id,
			source_bot=source_bot,
			hero_cards=hero_cards,
			board_cards=board_cards,
			status=status,
			timestamp=timestamp
		)
	except Exception as e:
		print(f"DynamoDB write error: {e}")
		return {
			'statusCode': 500,
			'body': json.dumps({'error': 'DynamoDB write failed', 'details': str(e)})
		}
    
	# Archive to S3 (raw snapshot history)
	try:
		write_to_s3(
			snapshot=snapshot,
			table_id=table_id,
			timestamp=timestamp
		)
	except Exception as e:
		print(f"S3 write error: {e}")
		# Non-fatal: DynamoDB write succeeded, S3 is just archival
    
	return {
		'statusCode': 200,
		'body': json.dumps({
			'ok': True,
			'table_id': table_id,
			'message': 'Snapshot processed'
		})
	}


def write_to_dynamodb(table_id, source_bot, hero_cards, board_cards, status, timestamp):
	"""Write latest table state to DynamoDB."""
	table = dynamodb.Table(DYNAMODB_TABLE)
    
	# Calculate TTL (auto-delete after 60s)
	ttl = int(timestamp) + TTL_SECONDS
    
	# Put item (overwrites existing)
	table.put_item(
		Item={
			'table_id': table_id,
			'source_bot': source_bot,
			'hero_cards': hero_cards,
			'board_cards': board_cards,
			'status': status,
			'last_seen': int(timestamp),
			'ttl': ttl
		}
	)
    
	print(f"DynamoDB: Updated {table_id}")


def write_to_s3(snapshot, table_id, timestamp):
	"""Archive raw snapshot to S3."""
	# Generate S3 key
	date = datetime.fromtimestamp(timestamp).strftime('%Y-%m-%d')
	key = f"snapshots/{table_id}/{date}/{int(timestamp)}.json"
    
	# Upload to S3
	s3_client.put_object(
		Bucket=S3_BUCKET,
		Key=key,
		Body=json.dumps(snapshot),
		ContentType='application/json',
		Metadata={
			'table_id': table_id,
			'timestamp': str(timestamp)
		}
	)
    
	print(f"S3: Archived {key}")


def validate_cards(cards):
	"""Validate card format: [rank][suit] where rank=A-9,T,J,Q,K and suit=h,d,c,s"""
	import re
	pattern = re.compile(r'^[A23456789TJQK][hdcs]$')
    
	for card in cards:
		if not pattern.match(card):
			return False
    
	return True
```

### 2.3 Lambda Function for GET /table/latest

**File:** `lambda_get_latest.py`

```python
import json
import boto3
from decimal import Decimal

dynamodb = boto3.resource('dynamodb')
DYNAMODB_TABLE = 'poker_tables'

def lambda_handler(event, context):
	"""
	Get all current tables from DynamoDB.
    
	Returns aggregated view of all active tables.
	"""
    
	table = dynamodb.Table(DYNAMODB_TABLE)
    
	try:
		# Scan all items (for small table, <1000 items)
		# For larger scale, use pagination or GSI query
		response = table.scan()
		items = response.get('Items', [])
        
		# Convert Decimal to float (DynamoDB returns Decimal)
		tables = [convert_decimals(item) for item in items]
        
		return {
			'statusCode': 200,
			'headers': {
				'Content-Type': 'application/json',
				'Access-Control-Allow-Origin': '*'  # CORS
			},
			'body': json.dumps({
				'ok': True,
				'table_count': len(tables),
				'tables': tables
			})
		}
        
	except Exception as e:
		print(f"DynamoDB scan error: {e}")
		return {
			'statusCode': 500,
			'body': json.dumps({
				'ok': False,
				'error': 'Failed to fetch tables',
				'details': str(e)
			})
		}


def convert_decimals(obj):
	"""Convert DynamoDB Decimal types to float for JSON serialization."""
	if isinstance(obj, list):
		return [convert_decimals(i) for i in obj]
	elif isinstance(obj, dict):
		return {k: convert_decimals(v) for k, v in obj.items()}
	elif isinstance(obj, Decimal):
		return float(obj)
	else:
		return obj
```

## 8. Cost Estimate

### 8.1 Monthly Costs (9 Bots, 1-2s Polling)

**Lambda Invocations:**
- 9 bots × 0.67 requests/second × 86400 seconds/day × 30 days = 15.6M requests/month
- First 1M free, then $0.20/1M = $2.92/month

**Lambda Duration:**
- 15.6M × 100ms avg × $0.0000166667/GB-sec (256MB) = $6.50/month

**DynamoDB:**
- On-Demand pricing (no capacity planning)
- Write requests: 15.6M × $1.25/1M = $19.50/month
- Read requests: Minimal (UI polls, not bots) = ~$1/month
- Storage: <1GB (9 tables, TTL cleanup) = $0.25/month

**S3:**
- PUT requests: 15.6M × $0.005/1K = $78/month
- Storage: ~1TB/month cumulative = $23/month (first month)
- After lifecycle (7 days → Glacier): ~$5/month

**Total Monthly Cost:**
- Lambda: $9.42
- DynamoDB: $20.75
- S3: $78 (PUT), $23 (storage first month)
- **Total: ~$131/month** (first month, decreases with Glacier)

**Compare to Flask on EC2:**
- t3.medium (24/7): $30/month
- EBS: $8/month
- **Total: $38/month**

**Trade-off:**
- Serverless: Higher cost ($131), zero maintenance, auto-scaling
- EC2 Flask: Lower cost ($38), manual scaling, server mgmt

**Optimization:**
- Reduce bot polling to 2s → halves Lambda/DynamoDB costs
- Aggressive S3 lifecycle → reduces storage costs
- **Optimized: ~$80/month**

### 5.1 Snapshot Collection Flow (Browser → Backend)

```
┌──────────────────┐
│   n4p.js         │  Injected into pokerbet.co.za iframe
│   (Browser)      │  Monitors DOM for table state changes
└────────┬─────────┘
		 │ Every 5s or on state change
		 │ POST /api/snapshot
		 │ {table_id, seat_index, snapshot}
		 ▼
┌──────────────────┐
│  Flask Backend   │  Receives snapshots
│  engine/app.py   │  Stores in _snapshot_store
└────────┬─────────┘
		 │ Merges snapshots per table
		 │ Builds _merged_tables
		 ▼
┌──────────────────┐
│  _merged_tables  │  Unified table state
│  (In-Memory)     │  9 seats with latest data
└──────────────────┘
```

### 5.2 Remote Control Flow (UI → Backend → Browser)

```
┌──────────────────┐
│  Remote UI       │  User clicks RAISE MAX button
│  index.html      │  Seat 3
└────────┬─────────┘
		 │ sendCommand(tableId, 3, 'raise_max')

		 ▼
┌──────────────────┐

		 │
		 ▼

@app.route('/api/commands/queue-batch', methods=['POST'])
def queue_batch_commands():
	"""Queue same command for multiple seats (MIN ALL / MAX ALL)"""
	payload = request.get_json()
	if not payload:
		return jsonify({'ok': False, 'error': 'No payload'}), 400

	table_id = payload.get('table_id')
	seat_indices = payload.get('seat_indices', [])
	command_type = payload.get('command_type')
	value = payload.get('value')

	if not all([table_id, seat_indices, command_type]):
		return jsonify({'ok': False, 'error': 'Missing required fields'}), 400

	# Validate command type
	try:
		validate_command_type(command_type)
	except ValueError as e:
		return jsonify({'ok': False, 'error': str(e)}), 400

	command_ids = []
	queued_count = 0

	with _store_lock:
		for seat_index in seat_indices:
			store_key = f"{table_id}:{seat_index}"
            
			# Check if seat exists and is active
			if store_key not in _snapshot_store:
				continue
            
			seat = _snapshot_store[store_key]
			if seat.get('status') != 'playing':
				continue

			# Generate token and queue command
			token = generate_seat_token(table_id, seat_index)
			command_id = str(uuid.uuid4())[:8]
            
			_command_queue[token] = {
				'id': command_id,
				'type': command_type,
				'value': value,
				'queued_at': time.time(),
				'status': 'pending',
			}
            
			command_ids.append(command_id)
			queued_count += 1

		# Refresh merged table
		if table_id in _merged_tables:
			_merged_tables[table_id] = merge_table(table_id)

	return jsonify({
		'ok': True,
		'command_ids': command_ids,
		'queued_count': queued_count
	})
```

@app.route('/api/commands/execute-all', methods=['POST'])
def execute_all_commands():
	"""Execute all pending commands (for BET / EXECUTE button)"""
	payload = request.get_json()
	if not payload:
		return jsonify({'ok': False, 'error': 'No payload'}), 400

	table_id = payload.get('table_id')
	if not table_id:
		return jsonify({'ok': False, 'error': 'Missing table_id'}), 400

	executed_count = 0
	failed_count = 0

	with _store_lock:
		# Find all pending commands for this table
		for token, cmd in list(_command_queue.items()):
			if cmd and cmd.get('status') == 'pending':
				# Check if this command belongs to this table
				# (token format: hmac(table_id:seat_index))
				# For now, mark all as executable
				cmd['status'] = 'executing'
				executed_count += 1

	return jsonify({
		'ok': True,
		'executed_count': executed_count,
		'failed_count': failed_count
	})
```

@app.route('/api/commands/cashout-all', methods=['POST'])
def cashout_all_seats():
	"""Emergency cashout - queue cashout for all connected seats"""
	payload = request.get_json()
	if not payload:
		return jsonify({'ok': False, 'error': 'No payload'}), 400

	table_id = payload.get('table_id')
	confirm = payload.get('confirm', False)

	if not table_id:
		return jsonify({'ok': False, 'error': 'Missing table_id'}), 400

	if not confirm:
		return jsonify({'ok': False, 'error': 'Must confirm cashout-all'}), 400

	cashed_out_seats = []

	with _store_lock:
		# Find all connected seats for this table
		for store_key, record in _snapshot_store.items():
			if record['table_id'] == table_id:
				seat_index = record['seat_index']
				token = generate_seat_token(table_id, seat_index)
				command_id = str(uuid.uuid4())[:8]

				_command_queue[token] = {
					'id': command_id,
					'type': 'cashout',
					'queued_at': time.time(),
					'status': 'pending',
				}

				cashed_out_seats.append(seat_index)

		# Refresh merged table
		if table_id in _merged_tables:
			_merged_tables[table_id] = merge_table(table_id)

	return jsonify({
		'ok': True,
		'cashed_out_seats': cashed_out_seats
	})
```

// Toggle pre-action (mutually exclusive)
function togglePreAction(seatIndex, actionType) {
  const checkFoldId = `check-fold-${seatIndex}`;
  const checkCallId = `check-call-${seatIndex}`;

  const checkFold = document.getElementById(checkFoldId);
  const checkCall = document.getElementById(checkCallId);

  // Mutual exclusivity
  if (actionType === 'check_fold' && checkFold.checked) {
	checkCall.checked = false;
	sendCommand(currentTableId, seatIndex, 'pre_action_check_fold', { enabled: true });
  } else if (actionType === 'check_call' && checkCall.checked) {
	checkFold.checked = false;
	sendCommand(currentTableId, seatIndex, 'pre_action_check_call', { enabled: true });
  } else {
	// Both unchecked, clear pre-action
	sendCommand(currentTableId, seatIndex, 'clear_preaction');
  }
}

	  } else {
		showWaiting();
	  }
	})

	  showWaiting();

#### 2.3 Glacier Instant Retrieval vs Standard Glacier

**Why Glacier Instant Retrieval?**
- Retrieval time: Milliseconds (vs 3-5 hours for Standard Glacier)
- Cost: $0.004/GB/month (vs $0.0036/GB for Standard Glacier)
- Use case: May need to reprocess old data quickly

**Trade-off:**
- Slightly more expensive than Standard Glacier ($0.0004/GB difference)
- For 100TB: Extra cost = $40/month

- Cost: $0.00099/GB/month (75% cheaper than Glacier Instant)

- **Additional savings: ~$300/month at 100TB scale**
- Only use if you NEVER need to reprocess old data quickly

## Risk Mitigation

### Risk 1: Data Loss (Self-Hosted PostgreSQL)

**Risk:** No automated RDS backups, rely on manual scripts

**Mitigation:**
- Daily automated backups to S3
- Retain 30 days of backups
- Test restore procedure monthly
- Monitor backup script execution
- Alert on backup failures

**Backup Redundancy:**
- S3 Standard-IA: Primary backup storage
- Glacier Deep Archive: Long-term retention (after 90 days)
- Multiple AZs in S3 (automatic)

### Risk 2: Performance Degradation

**Risk:** Self-hosted may be slower than managed RDS

**Mitigation:**
- Benchmark before/after migration
- Monitor query performance with pg_stat_statements
- Tune PostgreSQL config for workload
- Scale instance if needed (t3.medium → t3.large)

**Performance Targets:**
- Simple queries: <10ms (same as RDS)
- Join queries: <50ms
- Analytics: <500ms
- Dashboard: <100ms

**Monitoring:**
```sql
-- Weekly performance check
SELECT 
	query,
	mean_exec_time,
	calls
FROM pg_stat_statements
ORDER BY mean_exec_time DESC
LIMIT 10;
```

### Risk 3: Glacier Retrieval Delays

**Risk:** 30-day transition means more data in Glacier, slower access

**Mitigation:**
- Use Glacier Instant Retrieval (millisecond access)
- Not Glacier Deep Archive (12-48 hour delay)
- Budget for retrieval costs (~$3/month)
- Document retrieval procedures

**When You Need Old Data:**
```bash
# Glacier Instant Retrieval - immediate access
aws s3 cp s3://poker-raw-data/2026/03/01/snapshot.json . 
# Works immediately, no restore needed
```

### Risk 4: Instance Failure

**Risk:** Self-hosted on single EC2 instance (no HA)

**Mitigation:**
- Daily backups to S3 (restore in <30 minutes)
- EBS snapshots (automated weekly)
- Documented restore procedure
- Monitoring with CloudWatch alarms

**Recovery Time Objective (RTO):**
- Target: 30 minutes to restore from backup
- Maximum: 2 hours (including instance launch)

**Restore Procedure:**
```bash
# 1. Launch new instance (if needed)
aws ec2 run-instances --instance-type t3.medium ...

# 2. Download latest backup
aws s3 cp s3://poker-backups/postgres/latest.sql.gz .

# 3. Restore database
gunzip latest.sql.gz
psql -U poker_user -d poker_data < latest.sql

# 4. Restart Flask API
sudo systemctl restart flask-api

# 5. Verify
curl http://localhost:5000/api/health
```
