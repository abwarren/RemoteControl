GENT INSTRUCTION — SPLIT PUBLIC DASHBOARD + PRIVATE CONTROL PLANE
🎯 Objective

Deploy a public dashboard node (15.240.57.158) that serves UI via Nginx, while keeping all bot control + ingest endpoints private on MAIN_PROD.

No internal control ports must be exposed to the internet.

SECURITY RULES (MANDATORY)
1. MAIN_PROD Security Group

ALLOW:

TCP INTERNAL_API_PORT (e.g. 8080)
Source: sg-scrapers
Source: sg-dashboard (15.240.57.158)

DENY:

Any 0.0.0.0/0 on internal ports ❌

3. Scraper EC2 Security Group

ALLOW OUTBOUND:

TCP INTERNAL_API_PORT → MAIN_PROD private IP or SG

⚙️ STEP 1 — SSH INTO DASHBOARD NODE
ssh -i <key.pem> ubuntu@15.240.57.158
⚙️ STEP 2 — INSTALL NGINX
sudo apt update
sudo apt install -y nginx
sudo systemctl enable nginx
sudo systemctl start nginx
⚙️ STEP 3 — DEPLOY DASHBOARD FILES

Agent must:

# From local or MAIN_PROD
scp -i <key.pem> -r ./dashboard/* ubuntu@15.240.57.158:/var/www/dashboard/

Then on server:

sudo mkdir -p /var/www/dashboard
sudo chown -R ubuntu:ubuntu /var/www/dashboard
⚙️ STEP 4 — CONFIGURE NGINX (CRITICAL SPLIT)

Create config:

sudo nano /etc/nginx/sites-available/dashboard
NGINX CONFIG
server {
	listen 80;
	server_name 15.240.57.158;

	root /var/www/dashboard;
	index index.html;

	# PUBLIC UI
	location / {
		try_files $uri $uri/ =404;
	}

	# SAFE PUBLIC API (optional)
	location /api/public/ {
		proxy_pass http://MAIN_PROD_PRIVATE_IP:5000;
		proxy_set_header Host $host;
	}

	# 🚫 BLOCK ALL INTERNAL ROUTES
	location /api/internal/ {
		deny all;
		return 403;
	}

	location /bot-control/ {
		deny all;
		return 403;
	}
}

Enable:

sudo ln -s /etc/nginx/sites-available/dashboard /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
⚙️ STEP 5 — ENSURE MAIN_PROD BINDS TO PRIVATE INTERFACE

On MAIN_PROD:

Flask / API must NOT bind publicly

BAD ❌

app.run(host="0.0.0.0")

GOOD ✅ (or firewall restrict)

app.run(host="0.0.0.0", port=5000)

BUT enforce via SG:

only scraper SG + dashboard SG can reach it
⚙️ STEP 6 — UPDATE SCRAPER WORKERS

Replace any public endpoint:

OLD:
http://15.240.57.158:5000/api/ingest

NEW:
http://MAIN_PROD_PRIVATE_IP:5000/api/ingest

Workers must use private IP only.
