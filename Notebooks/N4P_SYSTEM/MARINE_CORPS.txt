Parallel Doctrine
PFS and PVT work in parallel.

PFS:
- builds the durable system core
- defines the production contract

PVT:
- builds speed-layer outputs around that core
- works on UI, analytics, helpers, and rapid feature loops

They do not compete.
They do not overwrite each other.
They converge through the Corporals.
Practical meaning
IF task = backend, infra, production deployment, schema, exact runtime logic
	→ assign to PFS

IF task = dashboard, interface, reporting, prototype tool, helper workflow
	→ assign to PVT

IF both touch the same feature
	→ PFS defines the contract
	→ PVT builds against it
	→ Corporals validate fit

Escalation logic
IF a lower-rank agent is blocked:
	escalate one level up

IF integration conflict exists:
	Corporal escalates to Lieutenant

IF mission priority conflict exists:
	Lieutenant escalates to Captain

IF task is clear and in-rank:
	execute immediately
